## Key Management

Turnkey provides enterprise-grade key management infrastructure built on [secure enclaves](/security/secure-enclaves). Store, recover, and manage cryptographic keys with programmable access controls, quorum approvals, and end-to-end encryption.

## Why Turnkey for Key Management?

Traditional key management forces tradeoffs between security, usability, and operational flexibility. Turnkey eliminates these tradeoffs:

- **Secure enclave storage**: Private keys are generated and stored inside hardware-backed secure enclaves. Key material never exists in plaintext outside the enclave boundary.
- **Programmable access controls**: Use the [policy engine](/concepts/policies/overview) to define who can access keys, under what conditions, and with how many approvals.
- **End-to-end encryption**: All key import and export operations use [HPKE encryption](/security/enclave-secure-channels) directly to the enclave. Your infrastructure never sees plaintext key material.
- **Quorum controls**: Require [multi-party approval](/concepts/users/root-quorum) for sensitive operations like key export or recovery.

## Core capabilities

**Key generation**: Create keys directly in secure enclaves. Supports ECDSA (secp256k1, P-256) and EdDSA (Ed25519) curves for blockchain and general cryptographic use.

**Key import**: Securely import existing keys from other providers or cold storage. Material is encrypted to the enclave before transmission.

**Key export**: Export keys with policy controls and quorum approval. Exports are encrypted to a target public key you control.

**Lifecycle management**: Rotate keys, manage access permissions, and maintain audit trails of all key operations.

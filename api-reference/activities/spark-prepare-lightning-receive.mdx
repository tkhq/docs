---
title: "Spark prepare Lightning receive"
description: "Generate a random payment preimage inside the Turnkey enclave, Feldman-split it for Spark Operators, and return only the payment hash — the raw preimage never leaves the enclave."
---

  <Expandable title="details">
    <NestedParam parentKey="parameters" childKey="signWith" type="string" required={true} default="">
    The Spark address of the receiving identity. Determines the enclave context in which the preimage is generated.
    </NestedParam>

    <ParamField body="lightningReceive" type="object" required={true} path="parameters.lightningReceive">
      <p>Configuration for the Lightning receive operation: how many Spark Operators share the preimage and who they are.</p>
      <Expandable title="lightningReceive details">
        <NestedParam parentKey="parameters.lightningReceive" childKey="threshold" type="number" required={true} default="">
          The Shamir secret sharing threshold — the minimum number of Spark Operators that must cooperate to reconstruct the preimage. Must be ≤ the number of `operatorRecipients`.
        </NestedParam>
        <ParamField body="operatorRecipients" type="array" required={true} path="parameters.lightningReceive.operatorRecipients">
          <p>The Spark Operators that will each hold an encrypted Feldman share of the preimage. When a Lightning payment arrives, threshold operators reconstruct the preimage and reveal it to the Spark Service Provider (SSP) to settle the payment.</p>
          <Expandable title="item details">
            <NestedParam parentKey="parameters.lightningReceive.operatorRecipients" childKey="operatorId" type="string" required={true} default="">
              The Spark Operator identifier (e.g. `"SO1"`).
            </NestedParam>
            <NestedParam parentKey="parameters.lightningReceive.operatorRecipients" childKey="encryptionPublicKey" type="string" required={true} default="">
              The operator's ECIES encryption public key, hex-encoded. The enclave uses this key to encrypt the operator's Feldman share so only that operator can decrypt it.
            </NestedParam>
          </Expandable>
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>

```bash title="cURL"
    "parameters": {
        "signWith": "<spark-address>",
        "lightningReceive": {
            "threshold": 2,
            "operatorRecipients": [
                { "operatorId": "SO1", "encryptionPublicKey": "<so1-pubkey-hex>" },
                { "operatorId": "SO2", "encryptionPublicKey": "<so2-pubkey-hex>" }
            ]
        }
    }
}'
```

```javascript title="JavaScript"
import { Turnkey } from "@turnkey/sdk-server";

const turnkeyClient = new Turnkey({
  apiBaseUrl: "https://api.turnkey.com",
  apiPublicKey: process.env.API_PUBLIC_KEY!,
  apiPrivateKey: process.env.API_PRIVATE_KEY!,
  defaultOrganizationId: process.env.ORGANIZATION_ID!,
});

const response = await turnkeyClient.apiClient().sparkPrepareLightningReceive({
  signWith: "<spark-address>",
  lightningReceive: {
    threshold: 2,
    operatorRecipients: [
      { operatorId: "SO1", encryptionPublicKey: "<so1-pubkey-hex>" },
      { operatorId: "SO2", encryptionPublicKey: "<so2-pubkey-hex>" },
    ],
  },
});

// Use the returned paymentHash to construct a BOLT11 invoice.
// The raw preimage never leaves the enclave.
console.log("Payment hash:", response.paymentHash);
```

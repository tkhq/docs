<H3Bordered text="Authorizations" />

<ParamField header="X-Auth-Proxy-Config-Id" type="string" required={true}>
  Your Auth Proxy config ID, found in **Dashboard → AUTH**. See [Auth Proxy reference](/reference/auth-proxy) for setup.
</ParamField>

<H3Bordered text="Body" />

<ParamField body="otpId" type="string" required={true}>

ID representing the result of an init OTP activity.
</ParamField>

<ParamField body="encryptedOtpBundle" type="string" required={true}>

Encrypted bundle containing the OTP code and a client-generated public key. Turnkey's secure enclaves will decrypt this bundle, verify the OTP code, and issue a new Verification Token. Encrypted using the target encryption key provided in the INIT_OTP activity result.
</ParamField>


<H3Bordered text="Response" />
A successful response returns the following fields:

<ResponseField name="verificationToken" type="string" required={true}>Verification Token containing a unique id, expiry, verification type, contact signed by Turnkey's enclaves. Verification status of a user is updated when the token is consumed (in OTP_LOGIN requests)</ResponseField>

<RequestExample>

```bash title="cURL"
curl --request POST \
  --url https://authproxy.turnkey.com/v1/otp_verify_v2 \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
  --header "X-Auth-Proxy-Config-Id: <string> (see Authorizations)" \
  --data '{
    "otpId": "<string>",
    "encryptedOtpBundle": "<string>"
}'
```

</RequestExample>

<ResponseExample>

```json 200
{
  "verificationToken": "<string>"
}
```

</ResponseExample>

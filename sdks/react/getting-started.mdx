## Installation

You can use `@turnkey/react-wallet-kit` in any React based web application.

For this guide, let's create a new `Next.js` app. If you already have an existing app, you don't need to do this.

```bash npx
npx create-next-app@latest
```

Now, install the Turnkey React Wallet Kit package:

<CodeGroup>

```bash npm
npm install @turnkey/react-wallet-kit
```

```bash pnpm
pnpm add @turnkey/react-wallet-kit
```

```bash yarn
yarn add @turnkey/react-wallet-kit
```

</CodeGroup>

<Info>

**Technical Requirements**

**React 19 Users**

If you're using React 19 you may encounter an installation error with `@turnkey/react-wallet-kit`. Consider:

- Downgrading React to `18.x.x`
- Using `npm install --force` or `--legacy-peer-deps`

</Info>

## Setup

<Steps>
 
  <Step title="Provider">
    Wrap your app with the `TurnkeyProvider` component, and import `"@turnkey/react-wallet-kit/dist/styles.css"` to include styles for the UI components.

    **Note** If you're using Next.js, this should be done in the `app/layout.tsx` file. The component **must** be a client component in order to add callbacks, so make sure to add `"use client";` at the top of the file.

    ```tsx src/app/layout.tsx
    "use client";

    import "./globals.css";
    import {
      TurnkeyProvider,
      TurnkeyProviderConfig,
    } from "@turnkey/react-wallet-kit";
    import "@turnkey/react-wallet-kit/dist/styles.css"; // Import styles. Crucial for using the UI components

    const turnkeyConfig: TurnkeyProviderConfig = {
      authProxyId: process.env.NEXT_PUBLIC_AUTH_PROXY_ID || "", // Your auth proxy ID from the dashboard
      organizationId: process.env.NEXT_PUBLIC_ORGANIZATION_ID || "",  // Your organization ID from the dashboard
    };

    export default function RootLayout({
      children,
    }: Readonly<{
      children: React.ReactNode,
    }>) {
      return (
        <html lang="en">
          <body>
            <TurnkeyProvider config={turnkeyConfig} >
              {children}
            </TurnkeyProvider>
          </body>
        </html>
      );
    }
    ```

    In case anything goes wrong, let's add an `onError` callback to the `TurnkeyProvider` to catch any errors that may occur.

    ```tsx src/app/layout.tsx
    <TurnkeyProvider
      config={turnkeyConfig}
      callbacks={{
        onError: (error) => console.error("Turnkey error:", error),
      }}
    >
    ```

  </Step>
</Steps>
